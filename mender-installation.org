* Context

This document provides a step-by-step guide for setting up *Mender OTA* (Over-The-Air) updates on NVIDIA Jetson devices and a companion workstation for artifact creation.

#+begin_quote
The recipe uses single-rootfs mode (no A/B partitions) suitable for JetPack installations.
All code blocks are designed to be idempotent where possible.
#+end_quote

** Target environment

| Component   | Target                                   |
|-------------+------------------------------------------|
| Device      | Jetson (JetPack 5.1, Ubuntu 20.04, arm64)|
| Workstation | Ubuntu 22.04 (WSL on Windows, amd64)     |
| Backend     | Mender Cloud (hosted.mender.io)          |
| Mode        | single-rootfs (no A/B)                   |

** System requirements

- NVIDIA Jetson Orin NX with JetPack 5.1+
- Ubuntu 22.04 workstation (or WSL) for artifact creation
- Internet connection on both systems
- Mender Cloud account with Tenant Token

* Phase 1: Jetson Device Setup (Ubuntu 20.04)

#+begin_quote
*RUN ON JETSON:* Open this file in Emacs on the Jetson device, or copy/paste
commands into a Jetson terminal. Blocks use =:dir /sudo::= for password prompts.
#+end_quote

** Step 1.1: Install Mender Client

Direct =.deb= downloads for Mender Client 3.x are no longer publicly accessible.
The only supported method on Ubuntu 20.04 is the official installer script.

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  curl -fLsS https://get.mender.io -o get-mender.sh
  sudo bash get-mender.sh mender-client
#+end_src

Verify installation:

#+begin_src bash :results output replace :exports both
  (dpkg -l | grep mender) 2>/dev/null || true
#+end_src

Expected output:
#+begin_example
ii  mender-client  3.5.x  arm64  Mender client for OTA software updates
#+end_example

** Step 1.2: Stop client before configuration

#+begin_src bash :results output replace :exports both :dir /sudo::
  sudo systemctl stop mender-client
#+end_src

** Step 1.3: Configure Mender Client for Mender Cloud

#+begin_quote
*ACTION REQUIRED:* Replace =<YOUR_TENANT_TOKEN>= with your actual Mender Cloud tenant token.
#+end_quote

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  # --- EDIT THIS ---
  TENANT_TOKEN="<YOUR_TENANT_TOKEN>"

  sudo tee /etc/mender/mender.conf > /dev/null <<CONFEOF
{
  "ServerURL": "https://hosted.mender.io",
  "TenantToken": "$TENANT_TOKEN",
  "InventoryPollIntervalSeconds": 300,
  "UpdatePollIntervalSeconds": 1800,
  "RetryPollIntervalSeconds": 300
}
CONFEOF

  echo "mender.conf written"
#+end_src

Validate JSON syntax:

#+begin_src bash :results output replace :exports both
  python3 -m json.tool /etc/mender/mender.conf || echo "Invalid JSON or file not found"
#+end_src

** Step 1.4: Configure single-rootfs mode

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  sudo mkdir -p /etc/mender/mender.conf.d

  sudo tee /etc/mender/mender.conf.d/single-rootfs.conf > /dev/null <<'CONFEOF'
{
  "RootfsPartA": "",
  "RootfsPartB": "",
  "BootUtilitiesSetActive": "",
  "BootUtilitiesGetNextActive": ""
}
CONFEOF

  echo "single-rootfs.conf written"
#+end_src

Validate:

#+begin_src bash :results output replace :exports both
  python3 -m json.tool /etc/mender/mender.conf.d/single-rootfs.conf || echo "Invalid JSON or file not found"
#+end_src

** Step 1.5: Ensure device identity exists

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  DEVICE_TYPE="jetson"

  if [ -f /var/lib/mender/device_type ]; then
    echo "device_type exists:"
    cat /var/lib/mender/device_type
  else
    echo "device_type=$DEVICE_TYPE" | sudo tee /var/lib/mender/device_type
    echo "Created device_type file"
  fi
#+end_src

** Step 1.6: Start the client

#+begin_src bash :results output replace :exports both :dir /sudo::
  sudo systemctl reset-failed mender-client
  sudo systemctl start mender-client
  (sudo systemctl status mender-client --no-pager -l | head -20) 2>/dev/null || true
#+end_src

Follow logs (run interactively):

#+begin_src bash :eval never :dir /sudo::
  journalctl -u mender-client -u mender-authd -u mender-updated -f
#+end_src

#+begin_quote
Required to build and deploy artifacts. Does *NOT* run on the Jetson.

** Step 2.1: Verify workstation OS

#+end_src

Expected:
#+begin_example
NAME="Ubuntu"
VERSION_ID="22.04"
#+end_example

** Step 2.2: Add Mender workstation-tools repository

Remove any incorrect repo first:

#+begin_src bash :results output replace :exports both :dir /sudo::
  sudo rm -f /etc/apt/sources.list.d/mender.list
  echo "Removed old mender.list (if any)"
#+end_src

Add GPG key:

#+begin_src bash :results output replace :exports both :dir /sudo::
  curl -fsSL https://downloads.mender.io/repos/debian/gpg \
    | sudo tee /etc/apt/trusted.gpg.d/mender.asc > /dev/null
  echo "GPG key added"
#+end_src

Add Ubuntu Jammy workstation repo:

#+begin_src bash :results output replace :exports both :dir /sudo::
  echo "deb [arch=amd64] https://downloads.mender.io/repos/workstation-tools ubuntu/jammy/stable main" \
    | sudo tee /etc/apt/sources.list.d/mender.list
#+end_src

Update package lists:

#+begin_src bash :results output replace :exports both :dir /sudo::
  (sudo apt update 2>&1 | tail -5) || true
#+end_src

** Step 2.3: Install workstation tools

#+begin_src bash :results output replace :exports both :dir /sudo::
  sudo apt install -y mender-artifact mender-cli
#+end_src

Verify:

#+begin_src bash :results output replace :exports both
  mender-artifact --version || echo "mender-artifact not installed"
  mender-cli --version || echo "mender-cli not installed"
#+end_src

* Phase 3: Test Deployment – File Update

This test validates the full OTA pipeline end-to-end using a minimal, non-invasive payload.

** Step 3.1: Create test file on workstation

#+begin_src bash :results output replace :exports both :session mender
  set -euo pipefail

  mkdir -p "$MENDER_TEST_DIR/files"
  echo "hello from mender $(date)" > "$MENDER_TEST_DIR/files/test.txt"

  echo "Created test file:"
  cat "$MENDER_TEST_DIR/files/test.txt"
#+end_src

** Step 3.2: Create file-based Mender artifact

#+begin_src bash :results output replace :exports both :session mender
  set -euo pipefail

  mender-artifact write module-image \
    -T "$DEVICE_TYPE" \
    -n test-file-deploy \
    -t file \
    -o "$MENDER_TEST_DIR/test-file-deploy.mender" \
    -f "$MENDER_TEST_DIR/files/test.txt"

  (ls -la "$MENDER_TEST_DIR"/*.mender) 2>/dev/null || true
#+end_src

** Step 3.3: Upload artifact to Mender Cloud

Authenticate (interactive, run manually):

#+begin_src bash :eval never
  mender-cli login
#+end_src

Upload:

#+begin_src bash :results output replace :exports both :session mender
  mender-cli artifacts upload "$MENDER_TEST_DIR/test-file-deploy.mender"
#+end_src

Verify:

#+begin_src bash :results output replace :exports both
  (mender-cli artifacts list | head -10) 2>/dev/null || true
#+end_src

* Phase 4: Test Deployment – Script Update

This test validates script-based OTA provisioning with package installation.

** Step 4.1: Purpose

Verify that Mender can:
- Execute scripts as root on the device
- Perform non-interactive package installation
- Modify user environment files
- Complete without reboot or A/B partitions

** Step 4.2: Prepare installation script

#+begin_src bash :results output replace :exports both :session mender
  set -euo pipefail

  mkdir -p "$MENDER_TEST_DIR/scripts"

  cat > "$MENDER_TEST_DIR/scripts/install-packages.sh" <<'SCRIPTEOF'
#!/bin/bash
set -e
export DEBIAN_FRONTEND=noninteractive

echo "[INFO] Starting package installation..."

# Add your package installation commands here
# Example:
# sudo apt-get update
# sudo apt-get install -y <package-name>

echo "[INFO] Package installation complete."
SCRIPTEOF

  chmod +x "$MENDER_TEST_DIR/scripts/install-packages.sh"
  echo "Script created:"
  (ls -la "$MENDER_TEST_DIR/scripts/") 2>/dev/null || true
#+end_src

** Step 4.3: Create script-based Mender artifact

#+begin_src bash :results output replace :exports both :session mender
  set -euo pipefail

  mender-artifact write module-image \
    -T "$DEVICE_TYPE" \
    -n install-packages-v1 \
    -t script \
    -o "$MENDER_TEST_DIR/install-packages-v1.mender" \
    -f "$MENDER_TEST_DIR/scripts/install-packages.sh"

  (ls -la "$MENDER_TEST_DIR"/*.mender) 2>/dev/null || true
#+end_src

** Step 4.4: Upload artifact to Mender Cloud

#+begin_src bash :results output replace :exports both :session mender
  mender-cli artifacts upload "$MENDER_TEST_DIR/install-packages-v1.mender"
#+end_src

Verify:

#+begin_src bash :results output replace :exports both

#+end_src

** Step 4.8: Notes on script-based deployments

- Scripts always run as root
- No interactive prompts allowed
- Package installs must use =-y=
- Scripts should be idempotent (safe to run multiple times)
- Failures cause deployment to be marked as failed

  sudo systemctl start mender-client

* Quick decision tree

- Device not appearing in Pending → check network, =journalctl -u mender-client=
- Authorization fails → verify TenantToken in mender.conf
- Deployment stuck → check =journalctl -u mender-client -f= on device
- Artifact upload fails → run =mender-cli login= first
- Script deployment fails → ensure script is executable and non-interactive
