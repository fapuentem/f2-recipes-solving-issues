* Context

This document provides a step-by-step guide for installing and configuring the *Mender OTA* client on NVIDIA Jetson devices.

#+begin_quote
The recipe uses single-rootfs mode (no A/B partitions) suitable for JetPack installations.
All code blocks are designed to be idempotent where possible.
#+end_quote

** Target environment

| Component | Target                                    |
|-----------+-------------------------------------------|
| Device    | Jetson (JetPack 5.1, Ubuntu 20.04, arm64) |
| Backend   | Mender Cloud (hosted.mender.io)           |
| Mode      | single-rootfs (no A/B)                    |

** System requirements

- NVIDIA Jetson Orin NX with JetPack 5.1+
- Internet connection
- Mender Cloud account with Tenant Token

* Step 1: Install Mender Client

Direct =.deb= downloads for Mender Client 3.x are no longer publicly accessible.
The only supported method on Ubuntu 20.04 is the official installer script.

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  mkdir -p /home/nvidia/Downloads
  curl -fLsS https://get.mender.io -o /home/nvidia/Downloads/get-mender.sh
  bash /home/nvidia/Downloads/get-mender.sh mender-client
#+end_src

Verify installation:

#+begin_src bash :results output replace :exports both
  (dpkg -l | grep mender) 2>/dev/null || true
#+end_src

Expected output:
#+begin_example
ii  mender-client  3.5.x  arm64  Mender client for OTA software updates
#+end_example

* Step 2: Stop client before configuration

#+begin_src bash :results output replace :exports both :dir /sudo::
  systemctl stop mender-client
#+end_src

* Step 3: Configure Mender Client for Mender Cloud

#+begin_quote
*ACTION REQUIRED:* Replace =<YOUR_TENANT_TOKEN>= with your actual Mender Cloud tenant token.
#+end_quote

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  # --- EDIT THIS ---
  TENANT_TOKEN="<YOUR_TENANT_TOKEN>"

  tee /etc/mender/mender.conf > /dev/null <<CONFEOF
{
  "ServerURL": "https://hosted.mender.io",
  "TenantToken": "$TENANT_TOKEN",
  "InventoryPollIntervalSeconds": 300,
  "UpdatePollIntervalSeconds": 900,
  "RetryPollIntervalSeconds": 300
}
CONFEOF

  echo "mender.conf written"
#+end_src

* Step 4: Configure single-rootfs mode

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  mkdir -p /etc/mender/mender.conf.d

  tee /etc/mender/mender.conf.d/single-rootfs.conf > /dev/null <<'CONFEOF'
{
  "RootfsPartA": "",
  "RootfsPartB": "",
  "BootUtilitiesSetActive": "",
  "BootUtilitiesGetNextActive": ""
}
CONFEOF

  echo "single-rootfs.conf written"
#+end_src

* Step 5: Ensure device identity exists

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  DEVICE_TYPE="f2-jetson-orin-nx"

  echo "device_type=$DEVICE_TYPE" | tee /var/lib/mender/device_type
  echo "Created device_type file"
#+end_src

* Step 6: Configure custom device identity

By default, Mender selects the Ethernet interface with the lowest =ifindex=, typically =eth0=.
If your devices use =eth1= or =eth2= for connectivity, override the identity script.

#+begin_src bash :results output replace :exports both :dir /sudo::
  set -euo pipefail

  mkdir -p /etc/mender/identity

  tee /etc/mender/identity/mender-device-identity > /dev/null <<'IDEOF'
#!/usr/bin/env bash
set -euo pipefail

# Interface priority (can override with IF_ORDER)
IFS_DEFAULT=("eth1" "eth2" "eth0")
read -r -a IF_ORDER <<< "${IF_ORDER:-${IFS_DEFAULT[*]}}"

is_valid_mac() {
  local mac="$1"
  local iface="$2"

  [[ "$mac" =~ ^([0-9A-Fa-f]{2}:){5}[0-9A-Fa-f]{2}$ ]] || return 1
  mac="${mac,,}"

  [[ "$mac" != "00:00:00:00:00:00" ]] || return 1
  [[ "$mac" != "00:00:00:00:00:01" ]] || return 1
  [[ "$mac" != "ff:ff:ff:ff:ff:ff" ]] || return 1

  # Skip virtual interfaces
  [[ -e "/sys/class/net/$iface/device" ]] || return 1

  return 0
}

for IF in "${IF_ORDER[@]}"; do
  addr_file="/sys/class/net/$IF/address"
  [[ -r "$addr_file" ]] || continue

  mac=$(<"$addr_file")

  if is_valid_mac "$mac" "$IF"; then
    echo "mac=${mac,,}"
    exit 0
  fi
done

echo "No valid MAC found on ${IF_ORDER[*]}" >&2
exit 1
IDEOF

  chmod +x /etc/mender/identity/mender-device-identity
  echo "mender-device-identity installed"
#+end_src

Test it:

#+begin_src bash :results output replace :exports both :dir /sudo::
  /etc/mender/identity/mender-device-identity
#+end_src

Expected output:
#+begin_example
mac=aa:bb:cc:dd:ee:ff
#+end_example

#+begin_quote
Changing the identity will cause the device to appear as a new device in Mender Cloud and require re-acceptance.
#+end_quote

* Step 7: Start the client

#+begin_src bash :results output replace :exports both :dir /sudo::
  systemctl reset-failed mender-client
  systemctl start mender-client
  (systemctl status mender-client --no-pager -l | head -20) 2>/dev/null || true
#+end_src

Follow logs (run interactively):

#+begin_example
journalctl -u mender-client -f
#+end_example

* Step 8: Accept device in Mender Cloud

#+begin_quote
*MANUAL STEP:* Complete in Mender Cloud UI.
#+end_quote

1. Navigate to *Devices* → *Pending*
2. Accept the device
3. Assign to static group: =jetson-test=
